# Website Update Guide (Internal)

This file is intentionally hidden (dotfile) and meant for maintainers.

## How this website is built

- Source code lives in `vue_project/` (Vue 3 + Vite + Tailwind + DaisyUI).
- The public site served from this repository root is a built snapshot:
  - `index.html`
  - `assets/*`
  - `vikischuster_website.svg`
- `vue_project/dist/` is generated output.
- Deployment currently works by copying `vue_project/dist/*` into the repository root.
- `vite.config.js` sets `base: '/website/'`, so this is configured for GitHub Pages under `/website`.

## Where to edit content (fast path)

- Hero text + links: `vue_project/src/components/Hero.vue`
- Section ordering/layout: `vue_project/src/App.vue`
- Projects list: `vue_project/src/assets/papers/projects.json`
- Papers list: `vue_project/src/assets/papers/papers.json`
- Talks/posters list: `vue_project/src/assets/papers/talks.json`
- Teaching list: `vue_project/src/assets/papers/teaching.json`
- Global styling: `vue_project/src/style.css`

## Standard update workflow

Fastest option (from repository root):

```bash
npm run publish
```

This command:

- builds the Vue app,
- clears old root `assets/` output,
- copies fresh `vue_project/dist/*` to repository root.

Manual option (if needed):

```bash
cd vue_project
npm install
npm run dev
```

Use the dev server to verify changes. Then publish:

```bash
npm run build
rm -rf ../assets
cd ..
cp -r vue_project/dist/* .
```

Commit and push:

```bash
git add index.html assets vikischuster_website.svg vue_project/src vue_project/package.json package.json .gitignore
# include any other files you changed

git commit -m "Update website content"
git push origin main
```

## Important rules

- Do not manually edit root `assets/index-*.js` or `assets/index-*.css` files.
  They are generated and will be replaced on next build.
- Always edit files in `vue_project/src/*` (or JSON content files) and rebuild.
- If only content changed, most edits are just JSON updates in `vue_project/src/assets/papers/*.json`.

## Simplified maintenance model (recommended)

Use this mental model:

1. Edit only in `vue_project/src/`.
2. Run `npm run publish` from root.
3. Commit source + generated root output.

This keeps edits clean and predictable while preserving current deployment behavior.
